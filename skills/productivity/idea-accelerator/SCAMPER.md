# SCAMPER Reference

Seven structured lenses for idea mutation. Apply each to the seed idea.

| Lens | Core question | Example mutation |
|---|---|---|
| **S — Substitute** | What component, material, person, or process can be replaced? | Replace human review with async AI pre-screening |
| **C — Combine** | What two separate things can be merged? | Combine onboarding + first meaningful action into one flow |
| **A — Adapt** | What from another context solves this problem? | Adapt subscription billing to per-token API pricing |
| **M — Modify / Magnify** | What happens if you make it bigger, faster, louder, more extreme? | Push the refresh cadence from daily to real-time |
| **P — Put to other uses** | Who else could use this? What else could this do? | A developer tool repurposed for non-technical ops teams |
| **E — Eliminate** | What happens if you remove a core element? What's the minimal version? | Remove the dashboard — deliver insights only via Slack DMs |
| **R — Reverse / Rearrange** | What if the sequence was flipped? What if the user did the agent's job? | Users curate data; AI handles presentation (reverse of default) |

## Usage

Work through all 7 in order. Each lens must produce at least one output variant. If a lens feels unproductive, spend 30 seconds on it and move on — don't skip.
